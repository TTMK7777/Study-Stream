## Phase 1 — 1 次試験基盤（✅ 完了 2026-05-09）

### Goal
中小企業診断士 1 次試験 7 科目をカバーし、ブラウザで誰でも使える Web アプリにする。

### スコープ
- **インフラ**
  - Next.js 16（App Router） + TypeScript 移行
  - API Routes で Anthropic API プロキシ化（API キー保護）
  - Supabase 導入（Auth + Postgres + RLS）
  - Vercel デプロイ
- **永続化**
  - ユーザー単位の学習履歴 / ハイライト / キャッシュ済みレッスン
  - 科目別進捗・正答率の蓄積
- **コンテンツ体系化**
  - 7 科目 × 各科目 10-30 論点のトピックツリー定義
  - `packages/content/` に YAML or TS で科目構造を宣言的に管理
  - 過去問頻出論点タグ（H30〜R5）
- **生成品質向上**
  - モデル更新（claude-sonnet-4-6 / claude-opus-4-7）
  - tool_use による structured output で JSON パース失敗を排除
  - prompt caching でコスト削減（同一トピック再生成時の system プロンプト固定）
- **UX 強化**
  - 科目選択 → 論点ツリー → レッスン の 3 段ナビ
  - 苦手論点フラグ（正答率 < 60% を自動マーク）

### 終了条件
- 7 科目全論点でレッスン生成が動作 ✅
- 認証ユーザーが履歴・ハイライトをデバイス跨ぎで参照可 ✅
- ベータユーザー 5 名で 1 週間継続使用 → 致命的バグなし（**Sprint 6 / 運用フェーズで検証**）

### 実装結果
- 12 PR（#10-#17, #23-#27、リカバリで #18-#22 は CLOSED）
- 93 テスト全 PASS
- 11 ルート（/, /login, /auth/callback, /api/lesson, /api/study-history, /api/highlights, /lesson/[topicId], /history, /highlights, /dashboard, /_not-found）
- 取締役会 + CISO レビュー反映（PR #26-#27）
- 各科目 5 論点（35 件）からスタート、Sprint 6 で 10-15 件へ

## Phase 2 — 2 次試験対応（+4-6 週）

### Goal
2 次試験の事例 I-IV に対応し、記述式 + AI 採点ループを実装。

### スコープ
- **事例文 UI**
  - 与件文（2000-3000 字）の段組み表示 + ハイライト
  - 設問1〜4 の入力欄（記述式、字数カウント、字数制限警告）
- **AI 採点エンジン**
  - 模範解答との比較プロンプト（観点: 切り口の一致 / キーワード抜け / 因果関係 / 文字数）
  - スコアリング（A/B/C/D）+ 改善コメント
- **過去問データセット**
  - H30-R5 の 6 年分 × 4 事例 = 24 セット（公開資料 + 自作模範解答）
- **答練モード**
  - 80 分タイマー、解答後一括採点

### 終了条件
- 24 過去問セットで答練 → AI 採点が完走
- 採点結果に対するユーザーの納得度（NPS 相当の主観評価）を計測
