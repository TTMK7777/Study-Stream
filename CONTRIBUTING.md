# コントリビューションガイド

StudyStream へのコントリビュートに興味を持っていただきありがとうございます。
本プロジェクトは [MIT ライセンス](LICENSE) の下で公開されているオープンソースソフトウェアです。
バグ報告、機能提案、ドキュメント改善、コード貢献など、あらゆる形の参加を歓迎します。

参加にあたっては [行動規範 (CODE_OF_CONDUCT.md)](CODE_OF_CONDUCT.md) の遵守をお願いします。

## Issue の報告

バグ報告や機能要望は GitHub Issue でお知らせください。以下の情報を含めていただけると対応がスムーズです。

### バグ報告
- 再現手順（できるだけ具体的に）
- 期待した挙動と実際の挙動
- 環境情報（OS、Node.js のバージョン、ブラウザ等）
- エラーメッセージやスクリーンショット（あれば）

### 機能要望
- 解決したい課題・ユースケース
- 想定する挙動や画面イメージ
- 代替案を検討した場合はその内容

## 開発環境のセットアップ

本プロジェクトは Next.js 16 + Supabase 構成です。

1. 依存パッケージをインストールします。

   ```bash
   npm install
   ```

2. `.env.example` をコピーして `.env.local` を作成し、環境変数を設定します。
   Supabase の URL / anon key / service_role key、`DATABASE_URL`、Anthropic の API キー / モデル名、
   `NEXT_PUBLIC_APP_URL` などのキーが必要です（具体的なキー名は `.env.example` を参照してください）。
   秘密情報は絶対にコミットしないでください。

3. 開発サーバーを起動します。

   ```bash
   npm run dev
   ```

4. テストを実行します（Vitest）。

   ```bash
   npm test
   ```

その他、`npm run build`（本番ビルド）、`npm run typecheck`（型チェック）、`npm run lint`（Lint）も利用できます。

## プルリクエストの流れ

1. `main` ブランチから作業ブランチを分岐します（`main` への直接 push は禁止です）。
2. 1 つの PR では目的を絞り、関連する変更のみを含めてください。
3. 変更前に `npm test` がすべて通ることを確認してください。
4. 必要に応じて `npm run typecheck` と `npm run lint` を実行し、緑であることを確認します。
5. わかりやすいコミットメッセージと PR 説明を添えて提出してください。

## コーディング規約

- 既存のコードスタイルを踏襲してください。
- 言語は TypeScript を使用します。型は可能な限り明示し、`any` の濫用は避けてください。
- Lint は `npm run lint`（ESLint / `eslint-config-next`）、型チェックは `npm run typecheck`（`tsc --noEmit`）で確認できます。
